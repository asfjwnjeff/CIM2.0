## 1. 概述

### 1.1 背景

CIM（客户信息管理系统）是面向半导体行业供应链服务的企业级管理平台。系统与COS（订单系统）、CPQ（报价计费系统）协作：

1. **COS下单**：客服录入字段 → CIM提供配置约束 → 字段标准化（下拉选择替代自由输入）
2. **CPQ计费**：读取CIM规则判断账单归属 → 每笔订单自动拆分到对应账单
3. **CIM售前报价**：支持销售快速出具客户报价 → 客户在线确认 → 合同签署后同步至CPQ正式计费

### 1.2 目标

- 建立系统级字段字典，支持灵活扩展分类和类型
- 按客户维度配置账单拆分字段及可选值
- 配置判断规则，实现订单到账单主体的自动匹配
- 完善风控审批流程，支持配置化和AI辅助（后续迭代）

### 1.3 范围

本次PRD覆盖以下模块：

**核心配置模块**：字典管理、账单拆分字段管理、账单主体规则管理
**核心业务模块**：客户管理、客户主体管理、商机管理、客户跟进、售前报价、合同签署、风控审批

## 3. 核心业务流程

### 3.1 账单拆分整体流程

```
COS订单系统                    CIM系统                      CPQ系统
┌────────────┐         ┌────────────────────┐        ┌────────────┐
│ 客服下单    │         │ 字典管理（IT配置）    │        │            │
│            │ ──────→ │                     │        │            │
│ 录入字段   │         │ 客户字段配置（客服配） │        │            │
│（标准化后）│         │         │            │        │            │
└────────────┘         │         ↓            │        │            │
                       │ 规则管理（客服配）    │        │            │
                       │         │            │        │            │
                       │         └────────────┼──────→ │ 自动计费    │
                       │                      │        │            │
                       │                      │ ←───── │ 生成账单    │
                       └────────────────────┘        └────────────┘
```

### 3.2 端到端业务主流程

```
客户创建 → 客户跟进（持续） → 商机建立（可选）
                                    ↓
                            售前报价（不强制绑定商机，客户直接询价即可创建）
                                    ↓
                            客户认可报价 → 客服二次确认
                                    ↓
                            绑定/创建商机 → 风控审批（必须绑定一条商机）
                               ↙      ↘
                          通过          驳回（修改后重新提交）
                            ↓
                        合同签署
                            ↓
                      CPQ正式报价录入 → 自动计费 → 生成账单
                            ↓
                        售后跟进（持续记录）
```

### 5.1 客户管理模块

**页面路径**: `/customers`
**使用角色**: 客服、客服主管、经理

#### 5.1.0 设计原则

CIM2.0 作为客户信息的**源头系统**，承担所有客户数据的创建和维护职责。

- **编辑权限全面开放**：标签1-7所有字段均可编辑（标签8-11为只读关联数据），新增与编辑使用相同的全字段表单

- **关键集成**：录入统一社会信用代码后自动触发企查查接口拉取工商资质信息；已存客户支持「刷新同步」手动拉取最新数据
- **协同操作**：支持协同（添加协同人）、分配（设置负责人）、移交（转移负责人）三种协同操作，列表页和详情页均可触发
- **跟进进度追踪**：客户详情页顶部展示6阶段进度步骤条，人工手动推进

**协同操作弹窗设计**（组件: `CollaborationDialogs`）:

> 负责人字段为 `responsiblePersons: string[]`，支持**多负责人**。

**协同弹窗**（添加协同人）:
- 标题: "添加协同人 — [客户名称]"
- 已选协同人列表（头像 + 姓名，每项右侧×移除按钮）
- 用户搜索/选择下拉框（多选，搜索系统用户，含防抖300ms）
- 确认/取消按钮
- 效果: 协同人获得该客户的查看和编辑权限，多人以追加方式添加

**分配弹窗**（追加负责人）:
- 标题: "设置负责人 — [客户名称]"
- 当前负责人列表显示（头像+姓名，可多选移除）
- 用户搜索/选择下拉框（多选，搜索系统用户）
- 确认/取消按钮
- 效果: 新负责人**追加**到 responsiblePersons 数组（不替换已有负责人），记录操作日志

**移交弹窗**（替换单个负责人）:
- 标题: "移交负责人 — [客户名称]"
- **步骤1**: 从当前负责人中选择"移交给谁"（下拉单选已有负责人，过滤掉新负责人）
- **步骤2**: 选择新负责人（下拉单选，不可选已存在的负责人）
- 移交原因（可选文本）
- 确认/取消按钮
- 效果: 将选中的老负责人替换为新负责人（responsiblePersons 数组中替换单个元素），被替换的原负责人自动变为协同人，操作日志记录移交详情

### 5.4 售前报价模块

**页面路径**: `/quotes`
**使用角色**: 经理/主管（创建编辑、定价、模板维护）、客服（查看、发送、确认）、客户（H5查看确认）

#### 5.4.1 模块概述

**背景**：CPQ 报价管理系统功能完整但录入复杂（三级产品层级、四种计费类型、四种计费模式、11种计费因子、起讫地/车型矩阵配置），不适合售前阶段频繁修改、快速响应的场景。售前报价定位为：**让客户快速看清费用怎么收，不做自动计费**。

**核心差异**：

| 对比维度 | CPQ 正式报价 | CIM 售前报价 |
|----------|-------------|-------------|
| 目标 | 正式计费、自动算费 | 快速出价、客户确认 |
| 复杂度 | 计费因子/计费模式/起讫地/车型/业务映射 | 费用项 + 单价 + 折扣 + 备注 |
| 绑定关系 | 必须走完整计费流程 | 不强制绑定商机/跟进记录 |
| 协同 | 无 | 经理→客服→客户→反馈→调价→新轮次 |
| 输出 | 进入计费→账单→对账→发票 | PDF版式导出 + H5链接分享 |
| 产品名称 | L1/L2/L3 标准目录 | 同样 L1/L2/L3 标准目录 |

**设计原则**：
1. 标准报价模板（类似菜单）→ 基于模板创建客户报价 → 调价后发送
2. 售前报价本身不强制绑定商机或跟进记录，客户直接询价即可创建
3. 费用名称必须遵循 L1/L2/L3 标准产品目录，不可自由输入
4. 售前只需展示单价，不需要数量和总价预估
5. 备注是核心字段，承载计费规则说明，同步至 CPQ
6. 客户接受报价后，走风控审批时**必须绑定一条商机**（如无则需先创建）

#### 5.4.2 业务模板库

**设计理念**：售前报价不是每个客户都从零写起。经理根据业务类型维护标准报价模板（类似菜单），新建客户报价时选择模板即可带入费用项和参考单价，再根据客户情况调整。

**模板按业务类型拆分**（非按客户分层）：
- 仓储报价模板
- 运输报价模板
- 关务报价模板
- 其他/增值服务报价模板

**模板选择规则**：新建报价时可多选模板 → 系统拼接选中模板的费用行，按业务分组展示。

**模板列表页**：

| 列 | 说明 |
|------|------|
| 模板名称 | 如"标准保税仓储报价模板" |
| 适用业务 | 仓储 / 运输 / 关务 / 其他 |
| 版本号 | 模板自身版本，如 V2.1 |
| 费用行数 | 模板内含费用项数量 |
| 维护人 | — |
| 最近更新 | — |
| 状态 | 启用 / 停用 |
| 客服可见 | 是/否 — 控制客服能否查看模板内的参考单价 |

**模板详情字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| 模板名称 | 文本 | — |
| 适用业务 | 单选 | 仓储/运输/关务/其他 |
| 费用明细 | 内嵌表格 | 每行：L1→L2→L3 + 计费单位 + 参考单价 + 备注 |

**模板操作**：

| 按钮 | 角色 | 说明 |
|------|------|------|
| 新建模板 | 经理 | 创建新模板 |
| 编辑 | 经理 | 修改费用项和参考单价 |
| 停用/启用 | 经理 | 停用后不可用于新建报价 |
| 复制 | 经理 | 基于当前模板复制 |

**客服权限规则**：
- 模板"客服可见"= 是 → 客服可查看参考单价
- 模板"客服可见"= 否 → 客服仅可查看费用名称和计费单位，不展示参考单价
- 理由：客服作为中转需要向客户解释价格如何收取，但底层模板的定价策略需要保护

| 是否含税 | 开关 | 是 | 含税 | — | 可编辑 |

##### 区块二：报价明细行（费用清单）

费用行从所选模板继承，按业务分组折叠展示。

| 字段 | 类型 | 必填 | 说明 | 可编辑 |
|------|------|------|------|--------|
| 业务分组 | 标签 | — | 由所选模板决定，按仓储/运输/关务/其他分组 | 否 |
| 产品L1（费用大类） | 只读 | — | 由 L3 选择后自动带出 | 否 |
| 产品L2（费用子类） | 只读 | — | 由 L3 选择后自动带出 | 否 |
| 产品L3（费用名称） | 下拉搜索 | 是 | 标准产品目录，单选 | 增行时可选 |
| 计费单位 | 下拉 | 是 | 标准枚举：票/件/个/项/料号/㎡/kg/km/车/人次/天/人/托/证/批次… | 是 |
| 单价 | 数值 | 是 | 继承模板参考价，可修改 | 是 |
| 行折扣 | 百分比 | 否 | 默认100%（无折扣），如输入95% | 是 |
| 行折扣后单价 | 数值 | — | 单价 × 行折扣，自动计算 | 否 |
| 备注 | 文本 | 否 | **核心字段**：计费规则说明，如"含HSN编码5条，超出+50元/条"、"min 300元/票" | 是 |

**行级操作**：
- 新增行：经理可在分组下添加费用项
- 删除行：经理可删除某行
- 上移/下移：调整费用行顺序

**费用名称选择规则**：
- 必须从标准产品目录中搜索选择，L1→L2→L3 三级级联
- 选择 L3 后，L1 和 L2 自动带出
- 不可自由输入文本 —— 保障后续 CPQ 同步时能匹配标准费用项

##### 区块三：报价汇总

| 字段 | 计算方式 |
|------|----------|
| 费用合计 | Σ(行折扣后单价) |
| 整单折扣 | 输入折扣率，如 95% |
| 报价总金额 | 费用合计 × 整单折扣 |
| 税额 | 含税时 = 报价总金额 × 税率 |
| 含税总金额 | 含税时 = 报价总金额 + 税额 |

**折扣双层级说明**：
- **行级折扣**：单独费用项的折扣（如"运输费打95折"）
- **整单折扣**：总价层面的折扣（如多轮商谈后客户说"总价打个95折"）
- 客户 H5 端展示折扣后价格，不展示折扣率

##### 区块四：操作日志与轮次快照

展示方式：抽屉面板/折叠区。

**日志记录内容**：
- 时间戳 + 操作人 + 操作类型（创建/修改/发起轮次/发送/客户确认/客服确认/状态变更）
- 字段级差异：字段名、改前值 → 改后值

**轮次快照**：
- 每发起一轮报价 = 生成一次完整快照
- 快照包含当时全部头信息 + 费用明细 + 汇总
- 可回溯查看历史任意轮次的完整内容

##### 区块五：操作按钮

| 按钮 | 可见角色 | 触发前提 | 点击后行为 |
|------|----------|----------|-------------|
| 保存草稿 | 经理 | — | 保存当前编辑，状态不变 |
| 发起报价 | 经理 | 头信息必填完整 + 至少一条费用行 + 有效期已设 | 二次确认："确认生成第N轮报价并发送？" → 轮次+1，生成快照，状态变为待发送（首轮）或保持报价中，自动生成 PDF 和 H5 链接 |
| 生成PDF | 经理、客服 | 已有轮次快照 | 按预设 PDF 模板渲染，下载 |
| 复制分享链接 | 经理、客服 | 已有轮次快照 | 生成/刷新当前轮次的 H5 链接，复制到剪贴板 |
| 调整价格 | 经理 | 状态=报价中 | 进入编辑模式，可修改单价/行折扣/整单折扣/备注/增删行 |
| 标记已接受 | 客服 | 客户在 H5 端已点击确认（客服二次确认） | 状态=已接受，记入操作日志。随后需绑定/创建商机方可发起风控审批 |
| 标记已拒绝 | 经理、客服 | — | 状态变更为已拒绝，记入操作日志 |
| 重新激活 | 经理 | 状态=已拒绝/已过期 | 恢复为报价中，延续当前内容编辑（不需新建轮次） |
| 同步至CPQ | 经理 | 状态=已接受 + 合同已签署 | 打包数据推送至 CPQ，状态标记"已同步" |
| 手动过期 | 经理、客服 | 状态=报价中 | 状态变更为已过期 |

**未保存离开提醒**：编辑态下切换页面或关闭浏览器时，弹出"尚有未保存的修改，是否保存？"

#### 5.4.5 客户 H5 报价查看页（P4）

**访问方式**：通过加密 shareToken 链接访问，无需登录。

**客户可见内容**：

- 报价汇总：费用合计、整单折扣（如有）、含税总金额

**客户不可见内容**：L1/L2、参考单价、行级折扣率、内部备注

**交互按钮**：

| 按钮 | 行为 |
|------|------|

**链接生命周期规则**：

| 规则 | 说明 |
|------|------|
| 有效期内可访问 | 跟随报价有效期，到期自动失效 |
| 新轮次触发旧轮次失效 | 经理发起新一轮后，旧轮次链接立即失效 |
| 失效展示 | 页面显示："该报价链接已失效，请联系客服获取最新报价" |
| 快照锁定 | 每个轮次的 H5 内容基于该轮快照固定不变，后续修改不影响已发送的轮次 |

| 整单折扣 | 报价汇总 | — |
| 含税/税率 | 头信息 | — |

| 报价轮次历史 | 确认轮次号、确认时间 | — |
| 备注 | **逐行备注** | 关键字段：计费规则说明，如"含5条HSN编码超出+50元/条"、"min 300元/票" |

**支持操作符**（8种）: 等于、不等于、包含、不包含、为空、不为空、在列表中、不在列表中

**匹配逻辑**: 按优先级从小到大依次评估，首次匹配即返回。规则总数约24条（分组归纳后）。

#### 5.10.4 新增/编辑规则

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 规则名称 | 文本 | 是 | |
| 所属客户 | 下拉选择 | 是 | 按客户维度配置 |
| 目标账单主体 | 下拉选择 | 是 | 按所选客户关联的账单主体筛选，下拉值域为该客户的账单主体可选值 |
| 优先级 | 数字 | 是 | 数字越小越优先，默认10 |
| 状态 | 开关 | 否 | 默认启用 |
| 备注 | 文本域 | 否 | 补充说明 |

**条件编辑器** (组件: `RuleGroupEditor`):

- **字段选择**: 下拉单选，按所选客户动态过滤。只显示该客户在账单拆分字段中配置的字段（排除账单主体）。应用材料→DN/Plant/Location，飞雅/荏原/苏斯/昇先创→客户部门，华力/岛津→货物属性。数据源来自 `initialCustomerBillingFields`
- **操作符选择**: 下拉单选，8 种操作符
- **值输入**:
  - 字段有配置可选值时 → **多选 checkbox 下拉**（选多个值，值间 OR 关系）
  - 字段无可选值时 → 文本输入框
  - 操作符为「为空/不为空」时 → 隐藏值输入
- **操作符自动转换**: 等于 + 多选 → 自动变为「在列表中」；不等于 + 多选 → 「不在列表中」；在列表中 + 单选 → 自动变回「等于」；不在列表中 + 单选 → 「不等于」
- **添加条件/子分组**: 支持嵌套 AND/OR 逻辑组合
- **账单主体下拉**: 可选值来自 `initialCustomerBillingFields` 中该客户「账单主体」字段的 options

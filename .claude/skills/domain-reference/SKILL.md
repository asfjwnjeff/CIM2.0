# 领域知识速查

涉及保险、业财税、客户管理领域时，优先使用行业术语，必要时向用户解释逻辑。

## 保险投保管理

- **核心流程**：投保→告知→核保（自动/人工）→签发→缴费→生效→保全/批改→退保/到期
- **关键实体**：投保人、被保人、受益人、险种、保额、保费、缴费计划、险种责任、条款
- **状态机**：草稿 → 待核保 → 核保中 → 核保通过/搁置/拒保 → 待缴费 → 已生效 → 已退保/已到期
- **核心字段**：投保单号、产品代码、险种代码、保额、保费、折扣系数、告知问卷、核保结论代码、缴费频率、首期/续期金额、保单状态、生效日、到期日、批改原因、批改生效时间

## 业财税一体化

- **核心流程**：报价→业务计费→生成账单→收款对账→开票→财务凭证
- **关键实体**：报价单、计费因子、费用项、账单、账单明细、收款单、发票、财务科目映射
- **状态流转**：报价中 → 已确认 → 已计费 → 账单已生成 → 部分收款/已收款 → 已开票 → 已入账
- **核心字段**：报价单号、客户ID、费率表版本、计费因子（数量/时长/规模）、单价、总金额、币种、账单周期、账单项类型、收款流水号、收款方式、核销状态、发票类型（普票/专票）、税率、发票状态、凭证号、财务科目编码、辅助核算项

## 客户信息管理

- **核心流程**：线索→客户→商机(必须)+服务产品→售前报价+方案管理→风控审批(绑定商机)→定制化报价修改→确定生效→[CIM结束→CPQ:合同签署→计费规则→自动计费→账单→开票→回款]
- **关键实体**：客户（大概念）、服务主体/签约主体/结算主体（三类独立主体，M:N 手动绑定）、商机、售前报价、方案、风控审批
- **关键概念**：
  - **售前报价**：售前阶段快速出价，让客户看清费用怎么收，不做自动计费；必须在商机下创建
  - **方案管理**：与报价并行，解决"怎么服务"；以合同物流服务方案书格式呈现
  - **服务产品**：9 项标准枚举（货代/关务/仓储/运输/进出口/维修/合同物流/一体化供应链/其他），贯穿商机→报价→方案→风控，链路上软约束一致
  - **报价轮次**：每次发送 = 一个快照轮次；轮内修改记日志不升轮次；新轮次触发旧 H5 链接失效
  - **CIM/CPQ 边界**：CIM 结束于报价生效；CPQ 负责合同签署及后续全部计费流程
- **核心字段**：客户ID、客户名称、证件类型/号、客户类型、行业、区域、数据来源、关联关系类型（控股/子公司/法人/实际控制人）、风险等级、KYC状态、黑名单标记、标签（多值）、跟进状态、商机阶段、服务产品、商机编号、报价轮次、行级折扣、整单折扣、L1/L2/L3 标准费用名称、计费单位、声明

## CIM2.0 系统业务规则

### 客户跟进状态自动流转

跟进进度由系统根据业务事件自动判断，不可手动修改（仅「失效」可手动设置）：

| 触发事件 | progressStatus | 说明 |
|----------|---------------|------|
| 客户新建（草稿） | `newly_acquired` 新获取 | 默认初始状态 |
| 客户首次提交 | `pending_followup` 待跟进 | status 从 draft → active |
| 创建商机（草稿或提交） | `preliminary_intent` 初步意向 | 仅当当前进度低于此级别时推进 |
| 风控审批通过 | `opportunity_confirmed` 商机确认 | 审批 status 变为 approved |
| 手动操作 | `invalid` 失效 | 仅此状态可手动更改 |

### 客户草稿与提交机制

- 新建客户/商机/跟进记录默认状态为 `draft`，风控审批已有草稿逻辑
- 草稿表单底部按钮：
  - **暂存** `[SaveIcon]`：灰色边框 `border-[#EBEBEB]` + 灰字 `text-[#5A5A5A]` + `rounded-xl`，不校验必填项，留在当前页面
  - **提交** `[PlusIcon]`：蓝色填充 `bg-[#2D3BFF]` + 白字 + `rounded-xl` + `shadow-sm`，校验后跳转
- 草稿客户详情页显示「提交」按钮，提交后 status → `active`，后续编辑显示「保存」
- 客户等级：K/A/B/C/D（表单可选值），不同等级对应不同跟进频率要求

### 跟进提醒通知

- **起算规则**：客户提交（status=active）后开始计时 → 每次写跟进记录以 `followUpDate` 重置起点
- **等级-天数映射**（可在系统设置中修改）：
  | 等级 | 天数 |
  |:--:|:--:|
  | K | 5 |
  | A | 10 |
  | B | 20 |
  | C | 30 |
  | D | 60 |
- **逾期判定**：今日 - 最后跟进日期 ≥ 等级天数 → 触发提醒
- **提醒对象**：客户的 `responsiblePersons` + `collaborators`
- **提醒展示**：AppLayout 右上角 Bell 红点 badge + 首页仪表盘黄色提醒卡片
- **配置入口**：系统设置 → 跟进提醒配置

### 半导体产业链定位条件显示

- **新建页**：仅当产业分类选择「半导体」时显示产业链层级/角色/标签字段
- **编辑/详情页**：半导体产业链定位为独立 Tab，仅当 `basicInfo.industryCategory === '半导体'` 时显示该 Tab

### 表单字段显示规则

- **新增/草稿页 vs 编辑/查看页**：新增时不显示的字段包括客户状态、客户代码、签约/服务/结算主体、预计月均业务量、仓库面积/温湿度、工商资质全景 Tab
- **服务产品**：单选（非多选），选「其他」时展开"其他服务产品需求"输入框（客户和商机均适用）
- **客户系统代码**：全局删除（与客户代码重复）

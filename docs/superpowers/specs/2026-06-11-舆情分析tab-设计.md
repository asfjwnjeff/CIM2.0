# 舆情分析 Tab — 设计方案

> 日期：2026-06-11 | 状态：设计已确认，待实施

## 一、背景与目标

在客户详情页新增「舆情分析」tab，通过智能体搜索客户相关的外部官方信息（官网新闻、监管公告、媒体报道、行业研报等），以两层结构展示：列表页 + 详情页。智能体采集功能由外部独立搭建，本方案聚焦前端页面框架、数据交互契约及 demo 数据。

## 二、数据模型

```typescript
interface SentimentItem {
  id: string;              // 唯一标识
  title: string;           // 标题
  source: string;          // 来源名称（东方财富、公司官网、证监会等）
  sourceType: string;      // 来源类型：官网公告 | 监管公告 | 媒体报道 | 行业研报
  publishDate: string;     // 发布日期
  summary: string;         // AI 摘要（100-200字）
  content: string;         // 正文全文
  images: string[];        // 图片 URL 列表
  originalUrl: string;     // 原文链接
  tags: string[];          // 关键标签（财报、高管变动、融资等）
  impactAssessment: string; // 影响评估：正面 | 中性 | 负面
  collectedAt: string;     // 系统采集时间
}
```

## 三、页面结构

### 3.1 列表页（Tab 内渲染）

**路由**：`/customers/[id]` 内 `sentiment` tab（不新建路由）

**布局**：

```
┌────────────────────────────────────────────┐
│  筛选栏                                     │
│  [来源类型▾] [影响评估▾] [日期范围] [🔍搜索] │
├────────────────────────────────────────────┤
│  📌 重点舆情（头条卡片）                      │
│  ┌──────────────────────────────────────┐  │
│  │ [大图] 标题 + 摘要 + 来源/时间         │  │
│  └──────────────────────────────────────┘  │
├────────────────────────────────────────────┤
│  舆情列表（紧凑式）                           │
│  ┌─ [缩略图] 标题 ─── 标签 标签 ────────┐  │
│  │  来源 · 日期                          │  │
│  ├──────────────────────────────────────┤  │
│  │  ...                                 │  │
│  └──────────────────────────────────────┘  │
├────────────────────────────────────────────┤
│            [ 加载更多 ]                      │
└────────────────────────────────────────────┘
```

**筛选能力**：
- 来源类型：官网公告 / 监管公告 / 媒体报道 / 行业研报
- 影响评估：正面 / 中性 / 负面
- 日期范围选择
- 关键词搜索

**分页**：初始加载 5 条，底部"加载更多"按钮每次加载后续条目

**点击条目** → 跳转到 `/customers/[id]/sentiment/[itemId]`

### 3.2 详情页

**路由**：`/customers/[id]/sentiment/[itemId]`（新建）

**布局**：左侧正文 + 右侧 AI 摘要面板

```
┌──────────────────────────────┐ ┌──────────────┐
│  来源类型 · 来源 · 发布日期    │ │ 🤖 AI 核心摘要 │
│                              │ │              │
│  标题（大号字体）              │ │ 一句话总结    │
│                              │ │ 关键数据      │
│  ┌────────────────────────┐  │ └──────────────┘
│  │      头图 / 封面图       │  │ ┌──────────────┐
│  └────────────────────────┘  │ │ 📊 影响评估    │
│                              │ │              │
│  正文全文...                  │ │ 😟 负面 92%  │
│                              │ │ 正面 8%      │
│  ┌────────────────────────┐  │ │ 中性 15%     │
│  │      正文插图           │  │ │ 负面 77%     │
│  └────────────────────────┘  │ └──────────────┘
│                              │ ┌──────────────┐
│  正文继续...                  │ │ 🏷️ 关键标签    │
│                              │ │ 财报 业绩下滑  │
│  ─────────────────────────  │ │ 机构评级 ...  │
│  🔗 查看原文    采集时间      │ └──────────────┘
└──────────────────────────────┘ ┌──────────────┐
                                 │ 📰 信息来源    │
                                 │ 来源/类型/日期 │
                                 └──────────────┘
```

**右侧 AI 面板内容**（280px 宽，固定）：

1. **AI 核心摘要**（蓝紫渐变卡片）：一句话总结 + 关键数据提炼
2. **影响评估**：情感结论 + 正/中/负百分比分布
3. **关键标签**：自动提取的主题标签列表
4. **信息来源**：来源名称、类型、发布日期、采集时间

## 五、数据接口契约

### `useSentiment(customerId: string)` Hook

```typescript
interface UseSentimentReturn {
  items: SentimentItem[];           // 当前筛选后的列表
  headline: SentimentItem | null;    // 头条（最新一条重要舆情）
  filters: SentimentFilters;
  setFilters: (f: Partial<SentimentFilters>) => void;
  hasMore: boolean;
  loadMore: () => void;
  getItemById: (id: string) => SentimentItem | undefined;
  isLoading: boolean;
}

interface SentimentFilters {
  sourceType: string | 'all';
  impactAssessment: string | 'all';
  dateRange: { start: string; end: string } | null;
  search: string;
}
```

将来接入真实智能体时，只需修改 `useSentiment` 内部实现，将 demo 数据替换为 API 调用，组件层无需改动。

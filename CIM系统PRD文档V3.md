## 1. 概述

### 1.1 背景

CIM（客户信息管理系统）是面向半导体行业供应链服务的企业级管理平台。系统与COS（订单系统）、CPQ（报价计费系统）协作：

1. **COS下单**：客服录入字段 → CIM提供配置约束 → 字段标准化
2. **CPQ计费**：读取CIM规则判断账单归属 → 每笔订单自动拆分到对应账单
3. **CIM售前报价**：销售快速出具报价 → 客户在线确认 → 合同签署后同步CPQ

### 1.2 目标

- 建立客户信息源头系统，管理客户全生命周期
- 联系人模块独立管理，支持多联系人+关键决策人标记
- 商机管理与客户联动，联系人从客户维度自动加载
- 跟进记录标准化，支持上门拜访打卡签到
- 风控审批流程精简，优化字段

### 1.3 范围

**核心业务模块**：客户管理、联系人管理、商机管理、跟进记录、售前报价、合同签署、风控审批
**核心配置模块**：字典管理、账单拆分字段管理、账单主体规则管理
**系统管理**：用户/角色/权限、跟进提醒配置

## 3. 核心业务流程

```
线索 → 客户(草稿→提交) → 联系人录入 → 商机创建 → 售前报价 → 风控审批
                                                    ↓
                                              合同签署(CPQ) → 计费 → 账单
```

**客户全生命周期**:
```
新建(草稿) → 提交(active) → 跟进中 → 商机确认 → 成交
                  ↓
               冻结/失效
```

### 5.2 客户管理模块

**页面路径**: `/customers`
**使用角色**: 客服、客服主管、经理

#### 5.2.0 设计原则

CIM2.0 作为客户信息的**源头系统**：
- 新建客户默认为**草稿(draft)**状态，提交后变为**活跃(active)**
- 编辑权限全面开放：标签1-7所有字段均可编辑
- 数据来源分区：企业基本信息=人工录入，工商资质信息=企查查拉取
- 联系人独立模块：Dialog弹窗管理，支持多联系人+关键决策人标记
- 跟进进度系统自动流转（仅「失效」可手动设置）

#### 5.2.1 草稿与提交机制

| 场景 | 按钮 | 行为 |
|------|------|------|
| 新建表单 | `[暂存]` `[提交]` | 暂存不校验、留在表单；提交校验后跳转列表 |
| 草稿记录再次打开 | `[暂存]` `[提交]` | 同上 |
| 已提交记录编辑 | `[保存]` | 校验后保存 |

**按钮样式**:
- 暂存: 灰色边框 `border-[#EBEBEB]` + 灰字 `text-[#5A5A5A]` + `rounded-xl`
- 提交/保存: 蓝色填充 `bg-[#2D3BFF]` + 白字 + `rounded-xl` + `shadow-sm`

#### 5.2.2 客户列表页

**操作栏**: 搜索框（客户名称/代码/信用代码/法定代表人/电话）、筛选（产业链层级/客户状态/跟进进度）、视图切换（卡片/列表）、"+ 新增客户"按钮

**批量操作工具栏**: 勾选后显示，支持批量协同/分配/移交

**列表视图列定义**:

| 列 | 内容 | 说明 |

#### 5.2.3 客户详情页（12个标签页）

**页面布局**: 顶栏(返回+名称+操作按钮) → 跟进进度步骤条 → 标签页导航 → 内容区

**跟进进度步骤条 (ProgressStepper)**:

| 值 | 标签 | 触发事件 |
|------|------|------|
| `newly_acquired` | 新获取 | 客户新建（默认） |
| `pending_followup` | 待跟进 | 客户首次提交(status: draft→active) |
| `preliminary_intent` | 初步意向 | 创建商机（仅当前级别低于此级时推进） |
| `opportunity_confirmed` | 商机确认 | 风控审批通过 |
| `deal_closed` | 成交 | 暂不实现 |
| `invalid` | 失效 | 手动操作（唯一可手动设置的状态） |

> **重要**: 跟进进度由系统自动判断流转，仅「失效」状态可手动更改。步骤条为只读显示，标注「系统自动判断」。

### 5.3 联系人管理模块 ⭐ 新增

**页面路径**: Dialog弹窗（客户详情/新建/编辑页中触发）
**使用角色**: 客服、客服主管、经理

#### 5.3.1 数据模型

联系人通过 `contacts` 表独立存储，通过 `customerId` 外键关联客户。每个客户可有多个联系人。

#### 5.3.2 交互设计

**Dialog布局**: 左右分栏（左260px + 右自适应）
- 左侧: 联系人列表（滚动）+ 「＋新增」按钮 + 计数
- 右侧: 选中联系人时显示16字段分组表单，未选中时显示引导文字

**表单分组**:
- 基本信息: 姓名、英文名、手机号、关键决策人、邮箱、微信
- 工作信息: 部门、职务、地址、邮政编码
- 个人信息: 性别、生日、年龄、爱好、籍贯、家庭情况

**操作按钮**:
- 编辑模式底部: [删除联系人]（红色边框） [取消] [保存]
- 新增模式底部: [取消] [保存]

**必填校验**: 姓名、部门、职务、性别、关键决策人为必填项，提交时校验。

#### 5.3.3 连通性

- **客户详情页**: 联系人以紧凑卡片列表展示，点击打开Dialog
- **商机新建/编辑页**: 联系人从所选客户自动加载，支持多选勾选
- **跟进记录**: 联系人从客户维度选择

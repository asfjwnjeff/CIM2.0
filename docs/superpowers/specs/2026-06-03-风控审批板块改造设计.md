# 风控审批板块改造设计

> 日期：2026-06-03
> 状态：待确认

## 1. 概述

对 CIM 2.0 风控审批板块进行全面改造，覆盖审批列表、审批流程配置、自动审批规则、审批字段配置四大模块及其交互关系，并为钉钉对接预留接口。

### 1.1 改造目标

- 建立以「服务产品」为中枢的审批体系：服务产品决定审批流程和动态字段
- 审批报告由自动规则引擎动态生成，编辑态实时刷新，审批中冻结
- 审批列表支持多维度筛选和自定义分组
- 预留钉钉审批对接接口

### 1.2 核心业务链路

```
创建审批 → 绑定商机+客户 → 选择服务产品
                              ↓
              ┌───────────────┴───────────────┐
              ↓                               ↓
      匹配审批流程（审批人链）        加载动态风控字段（表单）
              ↓                               ↓
         字段值可反哺流程                  用户填写
        （如贸易代理=是→加签）                ↓
              └───────────────┬───────────────┘
                              ↓
              编辑态：每次编辑→自动规则引擎→刷新报告
              审批中：报告冻结
              被驳回：恢复编辑态，继续自动刷新
                              ↓
              审批人按节点逐级审批，动态刷新状态
                              ↓
              未来：推送到钉钉审批，状态回传
```

## 2. 审批列表

### 2.1 数据权限

- 按客户归属控制可见范围：用户只能看到自己为「负责人」或「协同人」的客户的审批记录
- 负责人：可查看、编辑
- 协同人：仅查看
- 管理员（系统角色）：查看全部
- 注：完整权限模块（数据权限+功能权限）后期统一规划，本期仅定方向

### 2.2 默认展示

- 展示当前用户有权查看的所有审批记录
- 传统分页，底部页码

### 2.3 筛选

顶部三个搜索下拉框：

| 筛选项 | 说明 |
|--------|------|
| 审批状态 | 草稿 / 审批中 / 已通过 / 已驳回 |
| 服务产品 | 货代 / 关务 / 仓储 / 运输 / 进出口 / 维修 / 合同物流 / 其他 |
| 客户名称 | 搜索选择客户 |

### 2.4 快速分组 Tab

复用现有 GroupTabs 组件模式：

- **系统预设**（不可删除）：全部 / 当日新增 / 当月新增 / 我负责的 / 我协同的
- **自定义分组**：用户可创建，选字段 + 操作符 + 值，AND 逻辑组合条件
- 按账号隔离存储（localStorage）

### 2.5 视图模式

- 卡片视图 / 表格视图双模式切换

### 2.6 操作

| 操作 | 可执行状态 | 说明 |
|------|-----------|------|
| 新建 | 任意 | 创建新审批 |
| 查看详情 | 任意 | 查看审批详情、报告、流程 |
| 编辑 | 草稿 / 已驳回 | 修改审批信息，每次编辑触发报告刷新 |
| 撤回 | 审批中 | 撤回审批，回到编辑态 |
| 查看报告 | 任意 | 查看审批辅助报告 |
| 查看历史 | 任意 | 关键节点操作记录 |

### 2.7 历史记录

仅记录关键节点：

| 节点 | 记录内容 |
|------|---------|
| 提交 | 操作人 + 时间 |
| 通过（每个审批节点） | 审批人 + 时间 |
| 驳回 | 驳回人 + 时间 + 驳回原因 |
| 撤回 | 操作人 + 时间 |

## 3. 审批流程配置

### 3.1 配置权限

- 仅管理员可配置

### 3.2 流程定义

- 一个流程绑定一个服务产品
- 节点数量可自由配置
- 节点类型池（现有类型）：发起人 / 部门负责人 / 职能审批 / 财务 / 总经理 / IT 运维
- 每个节点手动指定审批人（具体人员，不依赖人事系统）
- 发起人不可修改流程，仅可添加抄送人（可选，自由选人）

### 3.3 界面布局

- 左侧：时间线预览审批链
- 右侧：表单编辑当前选中节点
- 保留现有混合布局模式

---

## 4. 自动审批规则

### 4.1 配置模式

- 多条件 + 多动作（保留现有模式）
- 条件之间 AND / OR 全局逻辑
- 字段来源：从「审批字段配置」表动态读取，不再硬编码

### 4.2 条件

- 每条条件：`[字段] + [操作符] + [值]`
- 操作符根据字段类型动态展示：
  - 数值型/百分比/数字：等于、不等于、大于、小于、大于等于、小于等于
  - 文本型/单选/多选：等于、不等于、包含、不包含
  - 布尔型：等于
- 多条条件用 AND / OR 组合

### 4.3 动作（4 种）

| 动作 | 说明 |
|------|------|
| 自动通过 | 满足条件时标记为通过 |
| 自动拒绝 | 满足条件时标记为拒绝 |
| 添加审批人 | 满足条件时在审批流中加签指定审批人 |
| 显示提示 | 满足条件时展示提示信息（含原因） |

> 已移除：跳过节点

### 4.4 执行时机

- **编辑态**（草稿/被驳回）：每次编辑后自动执行，刷新报告
- **审批中**：报告冻结，不再执行
- **被驳回**：恢复编辑态，恢复自动执行

### 4.5 界面布局

- 左侧表单 + 右侧实时预览

## 5. 审批辅助报告

### 5.1 内容结构

| 部分 | 内容 |
|------|------|
| 表头汇总 | 通过 N 条，风险提醒 M 条 |
| 明细列表 | 每条：规则名称 + 字段 + 判定结果（通过/风险）+ 原因 |

### 5.2 生命周期

- 编辑态：每次编辑后自动刷新
- 审批中：冻结
- 驳回后：恢复刷新

## 6. 审批字段配置

### 6.1 字段类型（7 种）

| 类型 | 说明 | 选项来源 |
|------|------|----------|
| 单选 | 可搜索下拉单选 | 管理员配置固定选项 |
| 多选 | 可搜索下拉多选 | 管理员配置固定选项 |
| 单选+其他 | 下拉 + 可填自定义值 | 管理员配选项 + 用户填"其他" |
| 布尔 | 是/否 | 固定 |
| 百分比 | 数字 + % 后缀 | - |
| 数字 | 仅数字，保留两位小数 | - |
| 文本 | 自由输入 | - |

### 6.2 字段绑定

- 一个字段可绑定多个服务产品
- 选择服务产品后，自动加载该产品绑定的所有字段
- 通用字段（所有产品共有）在字段配置中标记

### 6.3 必填控制

- 每个字段可配置是否必填
- 提交时校验

### 6.4 API 字段

- 客户信息、商机信息等从接口自动带出的字段，不进字段配置
- 系统自动填充，不可编辑

## 7. 模块间交互关系

```
字段配置 ──── 提供字段列表 ────→ 自动规则（条件中的字段下拉）
                  ↓
                提供动态表单字段 ←── 审批新建/编辑
                  ↓
服务产品 ──── 匹配审批流程 ────→ 审批流转
                  ↓
                匹配动态字段 ────→ 表单渲染
                  ↓
自动规则 ──── 读取字段值 ────→ 生成审批报告
                  ↓
审批报告 ←── 伴随审批实例 ────→ 审批列表展示
```

### 7.1 关键联动

| 联动 | 触发方 | 影响方 | 说明 |
|------|--------|--------|------|
| 服务产品 → 审批流 | 新建/编辑审批 | 流程匹配 | 选择产品后匹配对应工作流 |
| 服务产品 → 动态字段 | 新建/编辑审批 | 表单 | 选择产品后加载绑定字段 |
| 字段值 → 审批流 | 编辑态字段变更 | 流程加签 | 如贸易代理=是 → 自动添加审批人 |
| 字段配置 → 自动规则 | 字段增删改 | 规则条件 | 规则条件中的字段选项从字段配置读取 |
| 自动规则 → 审批报告 | 编辑态每次变更 | 报告刷新 | 规则引擎评估生成报告 |

## 8. 钉钉对接（预留接口，本期不开发）

### 8.1 接口预留

| 方向 | 接口 | 说明 |
|------|------|------|
| CIM → 钉钉 | 推送审批实例 | 发起审批时推送，含审批流程 + 报告 |
| 钉钉 → CIM | 审批结果回调 | 审批通过/驳回/转交等状态回传 |
| CIM → 钉钉 | 撤销审批实例 | 撤回时通知钉钉关闭审批 |

### 8.2 CIM 端展示

- 审批状态从钉钉回传后动态更新
- CIM 不提供审批操作（通过/驳回），仅展示状态

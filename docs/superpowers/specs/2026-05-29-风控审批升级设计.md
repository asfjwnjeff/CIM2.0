# CIM2.0 风控审批板块升级设计文档

**版本**: v1.1
**日期**: 2026-05-29
**状态**: 已完成（含验收修复轮次）

---

## 1. 背景与目标

### 1.1 背景

当前风控审批模块已实现基础功能（审批列表/新建/编辑/详情 + 流程配置 + 自动规则），但存在以下问题：

- 审批结构化字段硬编码在代码中，无法按服务产品动态配置
- 审批流模板审批人固定，缺乏可配置性
- 钉钉审批实例无法在字段上直接跳提示，缺乏审批辅助信息
- 审批流程配置页面存在数据源不一致、命名不一致等问题
- 新增字段影响审批流的扩展性不足

### 1.2 目标

- 审批结构化字段做成可配置，按服务产品自动带出
- 自动规则产出通过(绿)/提醒(黄)/拒绝(红)判断，生成审批辅助报告
- 审批辅助报告推送到钉钉附件 + CIM 独立查看
- 审批流模板可配置（6级默认节点 + 条件分支追加审批人）
- 规则可动态修改审批流（如贸易代理→自动追加白沥，多触发→会签）
- 减少人工审批工作量

## 2. 整体架构

三模块独立配置，通过数据关联：

```
审批字段配置 ──(定义字段)──→ 字段池
     ↓
服务产品 ←──(绑定字段)────── 字段池
     ↓
自动审批规则 ──(引用字段)──→ 条件匹配 ──→ 通过/提醒/拒绝
     ↓                                    ↓
审批流模板 ←──(规则可动态追加审批人)────────┘
     ↓
最终审批链 → 钉钉审批实例 + 审批辅助报告
```

### 2.1 提交审批完整流程

> 选择服务产品 → 加载对应结构化字段（用户填写）→ 自动规则运行 → 产出通过/提醒结果 → 生成审批辅助报告 → 规则动态修改审批流（追加审批人/会签）→ 匹配审批流模板确定审批链 → 推送钉钉

### 2.2 模块职责

| 模块 | 路由 | 职责 |
|------|------|------|
| 审批字段配置 | `/approval/fields`（新增） | 定义结构化字段池，绑定到服务产品 |
| 自动审批规则 | `/approval/auto-rules`（已有） | 引用字段做条件，产出通过/提醒/拒绝，可修改审批流 |
| 审批流模板 | `/approval/workflows`（改造） | 6级默认节点，按服务产品匹配审批人 |
| 审批辅助报告 | 无独立页面 | 提交时生成，钉钉附件 + CIM详情页Tab |

## 3. 模块一：审批字段配置（新增）

### 3.1 功能范围

新增独立配置页 `/approval/fields`，用于管理审批专用结构化字段。

### 3.2 页面布局

- **列表页**：表格展示（字段名称/Key/类型/绑定服务产品/状态/操作），支持搜索和按服务产品筛选
- **新增/编辑**：弹窗模式，包含字段名称、Key、类型、可选值配置、绑定服务产品、是否必填、状态、备注

### 3.3 字段类型支持

| 类型 | 说明 | 示例 |
|------|------|------|
| 数值（下拉） | 预设数值范围选项 | 0-5单 / 6-10单 / 11-20单 |
| 百分比 | 手动输入百分比 | 运输及时率 99% |
| 单选 | 预设选项列表，选择其一 | 恒温恒湿 / 常温 |
| 多选 | 预设选项列表，可选多个 | 意向服务城市 |
| 单选+其他 | 标准选项 + 其他(手工填写) | 运输产品种类 |
| 布尔 | 是/否切换 | 是否贸易代理 |
| 接口查询 | 调用外部接口自动获取 | 企查查-注册资本 |
| 文本输入 | 自由文本 | 备注说明 |

### 3.4 可选值配置

- 可选值仅需一个字段（标签），不做标签/值分离
- 支持增删改排序
- 规则引擎按文本做条件匹配

### 3.5 与服务产品关联

- 每个字段可绑定一个或多个服务产品
- 新建审批申请时，根据选择的服务产品自动加载对应字段

## 4. 模块二：自动审批规则（已有，保留不动）

### 4.1 当前状态

现有自动审批规则模块功能完善，包含条件+动作配置、AND/OR逻辑、实时预览。**本次升级中不修改此模块，后续测试后如有需要再单独出计划。**

### 4.2 升级后关联

- 规则条件中的可选字段从"审批字段配置"的字段池中引用
- 规则动作新增"追加审批人"类型，可指定目标审批人
- 多个规则触发追加不同审批人 → 职能审批节点变为会签

## 5. 模块三：审批流模板（改造）

### 5.1 功能范围

改造现有 `/approval/workflows` 页面。

### 5.2 默认审批节点（6级）

| 级别 | 节点类型 | 节点名称 | 性质 |
|------|----------|----------|------|
| 1 | initiator | 发起人 | 固定，无需配置 |
| 2 | department_manager | 部门经理 | 可配置审批人 |
| 3 | functional | 职能审批人 | **动态节点**（按服务产品匹配 + 规则追加） |
| 4 | finance | 财务会签 | 可配置审批人 |
| 5 | general_manager | 总经理 | 可配置审批人 |
| 6 | it_ops | IT运维确认 | 可配置审批人 |

### 5.3 核心交互

- **列表页**：表格展示（模板名称/服务产品/节点数/状态/更新时间），支持搜索筛选，支持复制模板
- **编辑页**：基本信息（名称/服务产品/状态/备注）+ 6级节点可视化配置
- **节点配置**：每个节点支持搜索添加/移除审批人
- **条件分支审批人**：职能审批节点预留给规则动态追加审批人的占位区域

### 5.4 模板匹配

按服务产品一对一匹配审批流模板。切换服务产品时审批流实时联动。

### 5.5 动态审批人 + 会签机制

- 职能审批节点默认审批人由服务产品决定（如运输→朱弢）
- 自动规则触发后动态追加审批人（如贸易代理→+白沥）
- 同一审批人被多个规则触发时仅出现一次（去重）
- 追加后职能审批节点审批人总数 > 1 时，自动变为会签模式（全部同意才通过）
- 每个追加审批人标注触发原因

## 6. 审批辅助报告

### 6.1 生成时机

提交审批时，规则引擎运行后自动生成。

### 6.2 报告格式

紧凑表格布局：
- 表头：审批点 / 字段 / 填写值 / 判断条件
- 通过行：左侧绿色 3px 色条 + 绿色圆点
- 提醒行：左侧黄色 3px 色条 + 浅黄底色 + 黄色圆点 + 建议文案
- 拦截行：左侧红色 3px 色条
- 顶部摘要：通过/提醒/拦截数量统计
- 底部：审批流变更摘要

### 6.3 查看入口

- **钉钉端**：作为审批实例附件推送
- **CIM端**：审批详情页新增「审批辅助报告」Tab

## 7. 审批申请页交互优化

### 7.1 页面布局

新建/编辑页保持双栏布局：
- **左侧**：表单区域（固定字段 + 动态结构化字段）
- **右侧**：审批流实时预览 + 实时规则检查

### 7.2 字段分组

所有字段按业务逻辑分为 4 组，每组左侧有色标：

| 分组 | 色标 | 包含内容 |
|------|------|----------|
| 基础信息 | 蓝色 | 公司全称、商机、服务产品、业务类型、风险控制目的等 |
| 业务评估 | 橙色 | 货物类型、月均业务量、月开票额、KPI要求等 |
| 合规审核 | 绿色 | 注册资本、社保人数、是否贸易代理等（动态字段） |
| 结算与合同 | 灰色 | HMG关系、业务主客户、开票信息、结算账期、联系人等 |

### 7.3 动态字段标识

- 会随服务产品变化的字段使用**浅蓝底色输入框**（`#F0F1FF` 背景 + `#C7CAFF` 边框）
- 固定字段保持白色背景
- "是否贸易代理=是"时该字段变为浅黄底色提示

### 7.4 交互亮点

- **选服务产品联动**：切换服务产品 → 左侧动态字段自动切换 → 右侧审批流实时匹配
- **实时规则检查**：填写字段后即时显示通过/提醒状态
- **条件分支可视化**：审批流上标注被追加的审批人及具体原因
- **提交前预览报告**：底部新增「预览报告」按钮
- **保存草稿**：支持分次填写

### 7.5 详情页审批进度

详情页右侧展示审批流当前进度：
- 已完成节点：绿色 ✓ + 绿色连线 + 完成时间
- 当前节点：蓝色光晕高亮 + 蓝色卡片，标注「审批中」
- 待处理节点：灰色描边 + 灰色文字
- 动态节点展示完整匹配信息（服务产品 → 审批人 + 规则触发分支）

## 9. 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 字段池模式 | 独立字段池（审批专用） | 与账单字段解耦，互不影响 |
| 架构模式 | 三模块独立配置（方案A） | 边界清晰，简单可扩展 |
| 审批流匹配 | 按服务产品一对一 | 当前场景足够，避免过度设计 |
| 选项值结构 | 标签即值（单字段） | 场景简单，双字段属多余 |
| 动态字段标识 | 浅蓝底色输入框 | 比小圆点更直观，不喧宾夺主 |
| 自动审批规则模块 | 保留不动，仅测试 | 当前功能完善，避免不必要的改动 |
| 审批报告格式 | 紧凑表格 + 色条 | 空间利用率高，一目了然 |
| 条件分支说明 | 写具体原因 | "涉及贸易代理自动增加白沥"比"条件触发"清晰 |

影响文件：approvals/new/page.tsx、approvals/[id]/edit/page.tsx、approvals/page.tsx

- `SERVICE_PRODUCT_COLORS`：服务产品颜色映射 Record（含 `'一体化供应链'` 和 `'其他'`）

# 调研过程标准模板

本文档定义调研过程中使用的所有标准表格格式和文档结构。SKILL.md 中引用此处模板。

---

## 1. 结构化选择题表格

每轮调研提出 2-4 个选择题时，使用此表格格式，降低用户回答成本：

```markdown
| # | 问题 | 选项 A | 选项 B | 选项 C | 用户选择 |
|----|------|--------|--------|--------|----------|
| 1 | 视图切换方式？ | 仅卡片 | 仅列表 | 两者可切换 | |
| 2 | 进度流转方式？ | 自动触发 | 人工手动推进 | — | |
| 3 | 协同操作交互？ | 单行操作 | 批量操作 | 两者都要 | |
```

**规范**：
- 每轮 2-4 个问题，不超过 4 个
- 选项 2-3 个，互斥且覆盖所有合理可能
- 用户选择后填入最后一列
- 如果某个问题只有 2 个选项，选项 C 填 "—"

---

## 2. 确认结论表

每轮对话结束时输出，作为该轮共识锚点：

```markdown
| 议题 | 结论 |
|------|------|
| 视图切换 | 客户列表页支持卡片视图（默认）和列表视图切换，切换时保持筛选条件 |
| 创建人字段 | 系统自动填入当前登录用户，新增时不可编辑 |
| 负责人字段 | 新增客户时必填，从系统用户列表中选择 |
```

**规范**：
- 每条结论简洁完整，脱离上下文也能理解
- 不用"是/否"，写成完整陈述句
- 作为后续调研记录中"确认结论"段落的正文

---

## 3. 文档产出表

每轮调研结束后，列出本轮产出/更新的所有文档：

```markdown
| 文档 | 更新内容 |
|------|----------|
| CIM系统PRD文档V2.md | v3.0.0 → v3.1.0：重写 §5.1 客户管理模块（§5.1.0-§5.1.3） |
| 客户管理结构化字段.md | v3.0.0 → v4.0.0：新增创建人/负责人/协同人/跟进进度字段 |
```

**规范**：
- 标注版本号变化
- 标注具体变更章节/字段，方便追溯
- 调研过程中随阶段推进持续更新此表
- 如果本轮无文档产出，写"本轮无文档变更"

---

## 4. 推翻/修正历史决策表

当本轮推翻了早期决策时，显式记录：

```markdown
| 历史决策（阶段） | 修正后 |
|-----------------|--------|
| 售前报价不绑定商机（第11.1轮） | 标准报价必须绑定商机 |
| 服务主体=客户本身，不需单独管理（第三阶段） | 服务主体独立管理，与客户 M:N 绑定 |
| 合同签署在CIM侧（第九阶段） | 合同模块整体移除，移至CPQ侧 |
```

**规范**：
- 左列写原决策内容 + 所在阶段号
- 右列写修正后的结论
- 被推翻的决策在调研记录正文中使用 `~~删除线~~` 标注
- 此表放在调研阶段的"确认结论"之后

---

## 5. 变更影响分析

当一次变更涉及 ≥3 个模块，或用户提供了新的流程图/架构图时，输出变更影响分析（替代 plan file）：

```markdown
### 变更影响分析

| # | 变更项 | 影响模块 | 影响文件 | 需回溯确认？ |
|----|--------|----------|----------|-------------|
| 1 | 合同模块移除 | 客户详情页、导航、PRD §5-§7 | CIM系统PRD文档V2.md | 是（第九阶段） |
| 2 | 三类主体重构 | 客户管理、主体管理 | 客户管理结构化字段.md | 是（第三阶段） |
| 3 | 方案管理新增 | 导航、售前报价流程 | CIM系统PRD文档V2.md | 否（全新模块） |
```

**规范**：
- 清单式，内嵌于调研记录，不另建独立文件
- 用户确认后即刻实施
- "需回溯确认"列标注是否需要回到历史阶段修正已确认结论

---

## 6. 用户原话引用格式

用户原始表述必须逐条引用，不加工、不概括：

```markdown
> "用户原话逐字引用，保留口语化表达和标点符号。"
```

**规范**：
- 使用 `> ` 引用块格式
- 原文保留，包括口语、缩写、不完整句子
- 每条引用独立成段
- 在每条引用之后，可加 1-2 句解读（非必须）

---

## 7. 调研记录文档结构

完整的调研过程记录文档按以下结构组织：

```markdown
# [项目名称] PRD调研过程记录

**版本**: v1.0
**日期**: YYYY-MM-DD
**说明**: 本文档记录了[项目名称]的完整调研过程，采用引导式设计方法，分步递进、逐步收敛。

---

1. 场景与角色梳理
2. 流程推演
3. 信息架构与字段补全
4. 交互与状态细化
5. 模块架构与边界重定义（多模块场景启用）

---

[简述为什么进入这个阶段]

[结构化选择题表格]

### 确认结论

|------|------|
| ... | ... |

### 文档产出
[文档产出表]

---

## 调研成果总结
```

## 8. 非文本输入识别清单

用户提供图片/Excel/.docx/PDF 后，必须首先输出此清单：

```markdown
### 从 [附件名称] 中识别到的内容

**文件类型**: [截图/Excel/流程图/Word/PDF]

**识别到的结构化内容**:

| # | 识别项 | 类型 | 识别内容 | 需确认？ |
|----|--------|------|----------|----------|
| 1 | 页面标题 | 文本 | "客户管理" | 否 |
| 2 | 字段：联系人 | 输入框 | 位于基本信息区 | 是 — 是否必填？ |
| 3 | 按钮：新增 | 操作 | 顶部工具栏 | 是 — 点击后行为？ |

**待确认项**: [列出所有标记"是"的项目，逐项向用户确认]
```

**规范**：
- 在开始引导式设计步骤之前输出
- 逐项标注准确性和是否需要用户确认
- 对于图片，描述视觉元素的位置关系（哪个区域、什么布局）

📋 核心实体说明

Customer (客户)

系统核心实体，存储客户的所有信息，包括基本信息、工商信息、产业链信息、关联企业、经营商品等。

PK: id

BillingEntity (账单主体)

账单拆分的目标主体，如FWD-8635、物流部等，是规则匹配的输出结果。

PK: id

BillingRule (账单拆分规则)

定义如何将订单拆分到不同账单主体的规则，支持嵌套AND/OR条件组。

PK: id

OrderMapping (订单映射)

记录订单与账单主体的匹配结果，包含拆分参数、匹配规则、操作人等信息。

PK: id

SplitField (拆分字段)

客户自定义的拆分字段配置，包括字段名称、类型、可选值等。

PK: id

SystemField (系统字段)

系统预定义的拆分字段模板，作为客户自定义字段的基础。

PK: id

RelatedCompany (关联企业)

客户的上下游关联企业，包括供应商、代工厂、采购商、合作伙伴等。

PK: id

CompanyProduct (经营商品)

客户经营的商品档案，包括商品名称、编码、报关要素、产地等。

PK: id

User (用户)

系统用户，包括管理员、操作员、查看者等角色。

PK: id

AuditLog (审计日志)

详细的字段级变更记录，记录谁在什么时间修改了什么字段的什么值。

PK: id

OperationLog (操作日志)

简化的操作记录，记录用户的主要操作行为。

PK: id

🔍 关系符号说明

||--o{ 一对多关系（一个客户拥有多个账单主体）
}o--|| 多对一关系（多个规则指向一个账单主体）
PK 主键（Primary Key）
FK 外键（Foreign Key）

🔄 关键业务流程

📦 账单拆分流程

1
COS 获取字段配置
COS 调用 CIM 接口获取客户的账单区分字段配置
↓
2
用户录入字段值
用户在 COS 中录入订单及账单区分字段值（Plant、Location、客户部门等）
↓
3
COS 发送订单到 CIM
COS 将订单号、客户ID、字段值发送给 CIM
↓
4
CIM 匹配规则
CIM 按优先级从高到低匹配 BillingRule，首次匹配成功即返回
↓
5
返回账单主体
CIM 返回匹配的 BillingEntity 给 COS
↓
6
记录映射结果
CIM 记录 OrderMapping，CPQ 可查询获取账单主体信息

⚙️ 规则配置流程

1
选择客户
在账单主体规则管理中选择要配置规则的客户
↓
2
创建规则
点击"新增规则"，填写规则名称和优先级
↓
3
配置条件组
添加条件组，支持嵌套 AND/OR 逻辑，设置字段、操作符、值
↓
4
选择目标主体
选择规则匹配成功后的目标账单主体
↓
5
保存并激活
保存规则并设置为激活状态，规则立即生效
